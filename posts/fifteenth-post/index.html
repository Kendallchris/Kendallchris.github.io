Automated Therapeutic Incentive Spirometer

Automated Therapeutic Incentive Spirometer

By: Chris Kendall

Date: 7 November 2024

Introduction

For my capstone project, my team collaborated with Lung Technologies to develop a modified therapeutic spirometer featuring an electronic monitoring system. This system is designed to enhance post-surgical lung therapy by automatically recording and displaying usage data, while also providing reminders to encourage compliance. The spirometer integrates mechanical and electronic subsystems to ensure accurate measurements, intuitive use, and effective therapy.

Problem Statement

Post-surgical patients often struggle to comply with spirometry therapy, leading to prolonged recovery and increased risks of complications like pneumonia. Traditional spirometers lack user-friendly features and automated data collection, placing a heavy burden on healthcare staff for monitoring and encouraging usage.

Key Features

The Automated Therapeutic Incentive Spirometer focuses on enhancing user compliance through the following features:

  • Data Recording and Feedback: An electronic module tracks lung function data and displays progress on an LCD screen.
  • Patient Reminders: Visual, auditory, and tactile reminders encourage regular use.
  • Accuracy and Reliability: Integrated sensors ensure precise measurements of lung capacity and functionality.
  • Ergonomic Design: Compact and lightweight, with easy-to-use controls, ensuring comfort for patients.

Design and Implementation

The system is powered by a central microcontroller, which coordinates inputs from sensors and outputs to provide feedback. It includes the following components:

Sensor Integration

  • Proximity Sensor: Tracks the position of the piston within the spirometer tube.
  • Motion Sensor: Detects user activity and wakes the system from sleep mode.
  • Accelerometer: Monitors device orientation, ensuring accurate readings.

Feedback System

  • LCD Screen: Displays real-time test results and progress updates.
  • Vibrator and Speaker: Provide tactile and auditory feedback to notify the user of their achievements or remind them to use the device.

Power Management

  • Lithium-Ion Battery: Supports three 8-hour sessions of active and standby use. Sleep mode minimizes power consumption during inactivity.

Coding Contributions

As the computer engineer on the team, I developed and implemented the code base, integrating the following functionalities:

  • Sensor Drivers: Managed data acquisition from proximity and motion sensors.
  • Feedback Algorithms: Processed sensor data to trigger reminders and provide real-time feedback.
  • User Interface Logic: Designed an intuitive interface for displaying test results and goal progress on the LCD screen.

    tft.setTextColor(TFT_WHITE, TFT_BLACK);
    tft.setTextSize(2);
    tft.setCursor(10, 10);
    tft.println("Waiting for object...");
    tft.setCursor(10, 40);
    tft.println("Press green button to cancel");
    Serial.println("Measurement mode activated");
}

// Function to enter sleep mode
void enterSleepMode() {
    Serial.println("Entering sleep mode...");
    tft.writecommand(TFT_DISPOFF);      // Turn off display
    tft.writecommand(TFT_SLPIN);        // Put display into sleep mode
    digitalWrite(ledPin, LOW);          // Turn off onboard LED
    isAsleep = true;                    // Set sleep mode flag
}

// Function to wake up from sleep mode
void wakeUp() {
    Serial.println("Waking up from sleep mode...");
    tft.writecommand(TFT_DISPON);       // Turn on display
    tft.writecommand(TFT_SLPOUT);       // Wake display from sleep
    isAsleep = false;                   // Clear sleep mode flag
    lastActivityTime = millis();        // Reset activity timer
    homeScreen.show(dataLogger.getCurrentHourMeasurements(true), dataLogger.getPreviousHourMeasurements());  // Return to home screen with current and previous measurements
}

Challenges and Solutions

  1. Battery Life Optimization: Balancing power demands of the sensors and feedback systems was critical. Sleep mode functionality and efficient coding practices ensured prolonged battery life.
  2. Measurement Accuracy: Sensor calibration and fluid dynamics analysis improved the precision of lung capacity readings.
  3. Patient Compliance: Implementing engaging reminders and an easy-to-read interface increased usability and compliance rates.

Conclusion

The Automated Therapeutic Incentive Spirometer represents a significant advancement in post-surgical respiratory therapy. By integrating intuitive electronics and reliable sensors, this device reduces the burden on healthcare staff while empowering patients to take control of their recovery.

The final design reflects engineering principles of modularity, reliability, and user-centered design. This project highlights how interdisciplinary collaboration can address real-world healthcare challenges, bridging the gap between medical needs and engineering innovation.
